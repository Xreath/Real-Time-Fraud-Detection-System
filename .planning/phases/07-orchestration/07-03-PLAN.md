**DAG configuration:**
```python
from airflow import DAG
from airflow.operators.python import PythonOperator, BranchPythonOperator
from airflow.operators.empty import EmptyOperator
from datetime import datetime, timedelta

from utils.alerting import on_failure_callback, write_alert, write_canary_alert
```

**Task functions (defined as module-level functions):**

1. `generate_data(**kwargs)`:
   - Get `num_transactions` from Airflow Variable (default 30000, per D-02): `from airflow.models import Variable; num_transactions = int(Variable.get("retraining_num_transactions", default_var=30000))`
   - Generate unique seed from run_id: `seed = int(hashlib.md5(kwargs["run_id"].encode()).hexdigest()[:8], 16) % (2**31)` (per D-01)
   - Call `from src.training.prepare_data import generate_historical_data; df = generate_historical_data(num_transactions=num_transactions, seed=seed)`
   - Save to temporary parquet: `df.to_parquet("/opt/airflow/data/retraining_raw.parquet", index=False)`
   - Push seed and num_transactions to XCom: `kwargs["ti"].xcom_push(key="seed", value=seed); kwargs["ti"].xcom_push(key="num_transactions", value=num_transactions)`
   - Print: `f"[1/8] Generated {num_transactions} transactions with seed={seed}"`

2. `check_data_quality(**kwargs)`:
   - Load from parquet: `df = pd.read_parquet("/opt/airflow/data/retraining_raw.parquet")`
   - Call `result = run_data_quality_checks(df)`
   - If not result["passed"]: write alert via `write_alert(alert_type="data_quality_failure", message="Data quality checks failed", dag_id=DAG_ID, metrics={"checks": result["checks"]})` and `raise ValueError("Data quality checks failed — blocking retraining")`
   - Print: `f"[2/8] Data quality: {'PASS' if result['passed'] else 'FAIL'}"`

3. `run_feature_engineering(**kwargs)`:
   - Load raw: `df = pd.read_parquet("/opt/airflow/data/retraining_raw.parquet")`
   - Call `from src.training.prepare_data import compute_features_with_spark; df_features = compute_features_with_spark(df)`
   - Save: `df_features.to_parquet("/opt/airflow/data/retraining_features.parquet", index=False)`
   - Print: `f"[3/8] Feature engineering complete: {len(df_features)} rows, {len(df_features.columns)} columns"`

4. `split_and_save(**kwargs)`:
   - Load features: `df = pd.read_parquet("/opt/airflow/data/retraining_features.parquet")`
   - Call `from src.training.prepare_data import split_data, save_splits; train_df, val_df, test_df = split_data(df); save_splits(train_df, val_df, test_df)`
   - Print: `f"[4/8] Data split: train={len(train_df)}, val={len(val_df)}, test={len(test_df)}"`

5. `train_models(**kwargs)`:
   - Set MLflow tracking URI: `import mlflow; mlflow.set_tracking_uri(os.getenv("MLFLOW_TRACKING_URI", "http://mlflow:5000"))`
   - Set experiment: `mlflow.set_experiment("fraud-detection-retraining")`
   - Call `from src.training.train_model import load_data, preprocess, get_models, train_and_log_model`
   - `train_df, val_df, test_df = load_data()`
   - `X_train, X_val, X_test, y_train, y_val, y_test, feature_names, scaler, label_encoders = preprocess(train_df, val_df, test_df)`
   - `pos_weight = (y_train == 0).sum() / max((y_train == 1).sum(), 1)`
   - `models = get_models(pos_weight)`
   - `results = [train_and_log_model(name, model, X_train, y_train, X_val, y_val, X_test, y_test, feature_names) for name, model in models]`
   - `from src.training.train_model import register_best_model; best = register_best_model(results)`
   - Push to XCom: `kwargs["ti"].xcom_push(key="best_model", value=best)`
   - Print: `f"[5/8] Training complete. Best: {best['name']} (Val F1={best['val_f1']:.4f})"`

6. `canary_evaluate(**kwargs)`:
   - Per D-05: Simulated canary via batch validation.
   - Get best_model from XCom: `best = kwargs["ti"].xcom_pull(key="best_model", task_ids="train_models")`
   - Get current champion metrics: `champion_metrics = get_champion_metrics()`
   - If no champion exists (empty dict), promote unconditionally.
   - Build challenger metrics from best_model result: `challenger_metrics = {"f1": best["val_f1"], "auc_pr": best["val_auc_pr"]}`
   - Note: Need to also get precision/recall from MLflow run. Use: `client = get_mlflow_client(); run = client.get_run(best["run_id"]); challenger_metrics["precision"] = run.data.metrics.get("val_precision", 0.0); challenger_metrics["recall"] = run.data.metrics.get("val_recall", 0.0)`
   - Save previous champion version for rollback reference (per CNRY-03): `prev_version = get_previous_champion_version()`
   - Call `result = compare_models(champion_metrics, challenger_metrics, threshold=0.95)`
   - Log canary evaluation as dedicated MLflow run (per specifics in CONTEXT.md):
     ```python
     mlflow.set_tracking_uri(os.getenv("MLFLOW_TRACKING_URI", "http://mlflow:5000"))
     mlflow.set_experiment("fraud-detection-retraining")
     with mlflow.start_run(run_name="canary-evaluation"):
         mlflow.log_params({"champion_version": champion_metrics.get("version", "none"), "challenger_run_id": best["run_id"], "threshold": 0.95})
         for k, v in result["comparison"].items():
             if isinstance(v, (int, float)):
                 mlflow.log_metric(k, v)
         mlflow.log_param("decision", "promote" if result["should_promote"] else "rollback")
     ```
   - Push to XCom: `kwargs["ti"].xcom_push(key="canary_result", value=result); kwargs["ti"].xcom_push(key="prev_champion_version", value=prev_version)`
   - Return branch: `return "promote_model" if result["should_promote"] else "rollback_alert"`
   - Print: `f"[6/8] Canary evaluation: {'PROMOTE' if result['should_promote'] else 'ROLLBACK'}"`

7. `promote_model(**kwargs)`:
   - Get best_model and canary_result from XCom
   - Get the new registered version: need to find the version for best model's run_id. Use: `client = get_mlflow_client(); versions = client.search_model_versions(f"run_id='{best['run_id']}'"); new_version = versions[0].version if versions else None`
   - If new_version: `promote_to_champion(MODEL_NAME, new_version)`
   - Re-register as pyfunc: `from src.serving.register_pyfunc import register_pyfunc_model; register_pyfunc_model()`
   - Write promotion alert: `champion_metrics = kwargs["ti"].xcom_pull(key="canary_result", task_ids="canary_evaluate"); write_canary_alert(action="promoted", champion_metrics=get_champion_metrics() or {}, challenger_metrics={"f1": best["val_f1"]}, champion_version=kwargs["ti"].xcom_pull(key="prev_champion_version", task_ids="canary_evaluate") or "none", challenger_version=new_version or "unknown")`
   - Print: `f"[7a/8] Model promoted to champion: v{new_version}"`

8. `rollback_alert(**kwargs)`:
   - Get canary_result, best_model, prev_champion_version from XCom
   - Per D-07: rollback means "don't promote" — champion alias stays at previous version
   - Write rollback alert with side-by-side comparison: `write_canary_alert(action="rolled_back", champion_metrics=..., challenger_metrics=..., champion_version=prev_version, challenger_version="not-promoted")`
   - Print: `f"[7b/8] Canary rollback — champion stays at v{prev_version}"`

9. `restart_fraud_api(**kwargs)`:
   - Per D-09: Use Docker Python SDK
   ```python
   import docker
   client = docker.from_env()
   container = client.containers.get("fraud-api")
   print(f"Restarting fraud-api container...")
   container.restart(timeout=30)
   print(f"fraud-api container restarted successfully")
   ```
   - If restart fails, exception propagates to on_failure_callback (per D-12)

10. `verify_health(**kwargs)`:
    - Per D-11: Hit health endpoint after restart
    ```python
    import time
    import requests
    max_retries = 10
    for i in range(max_retries):
        try:
            resp = requests.get("http://fraud-api:5002/health", timeout=5)
            if resp.status_code == 200:
                print(f"[8/8] fraud-api health check passed on attempt {i+1}")
                return
        except Exception as e:
            print(f"Health check attempt {i+1}/{max_retries}: {e}")
        time.sleep(10)
    raise RuntimeError("fraud-api health check failed after all retries")
    ```

**DAG task graph:**
```python
with DAG(DAG_ID, default_args=default_args, ...) as dag:
    t_generate = PythonOperator(task_id="generate_data", python_callable=generate_data)
    t_quality = PythonOperator(task_id="check_data_quality", python_callable=check_data_quality)
    t_features = PythonOperator(task_id="run_feature_engineering", python_callable=run_feature_engineering)
    t_split = PythonOperator(task_id="split_and_save", python_callable=split_and_save)
    t_train = PythonOperator(task_id="train_models", python_callable=train_models)
    t_canary = BranchPythonOperator(task_id="canary_evaluate", python_callable=canary_evaluate)
    t_promote = PythonOperator(task_id="promote_model", python_callable=promote_model)
    t_rollback = PythonOperator(task_id="rollback_alert", python_callable=rollback_alert)
    t_restart = PythonOperator(task_id="restart_fraud_api", python_callable=restart_fraud_api)
    t_verify = PythonOperator(task_id="verify_health", python_callable=verify_health)
    t_end = EmptyOperator(task_id="end", trigger_rule="none_failed_min_one_success")

    # Linear pipeline up to canary
    t_generate >> t_quality >> t_features >> t_split >> t_train >> t_canary

    # Branch: promote or rollback
    t_canary >> t_promote >> t_restart >> t_verify >> t_end
    t_canary >> t_rollback >> t_end
```

Important notes:
- Use `provide_context=True` is NOT needed in Airflow 2.x — kwargs are passed automatically
- The BranchPythonOperator in `canary_evaluate` returns the task_id string to execute next
- `t_end` uses `trigger_rule="none_failed_min_one_success"` so it runs after either branch
- The rollback path does NOT restart fraud-api (no new model was deployed)
